# Secrets Management

This repository supports [1Password](https://1password.com) integration via chezmoi's built-in template functions.

## Setup

1. Install the [1Password CLI](https://developer.1password.com/docs/cli/get-started/) (included in `packages.toml`):

   ```bash
   brew install --cask 1password-cli
   ```

2. Enable 1Password during `chezmoi init` when prompted (or set `use_1password = true` in your chezmoi config).

3. Sign in to 1Password CLI:

   ```bash
   op signin
   ```

## 1Password App — Manual Configuration

The following settings **must be enabled manually** in the 1Password desktop app. They are protected by cryptographic signing and cannot be automated via CLI or config files.

### Settings > Developer

> **Enable these in order** — "Show Developer Experience" reveals the rest.

| # | Setting | Purpose |
|---|---------|---------|
| 1 | **Show Developer Experience** | Reveals developer-focused settings in 1Password |
| 2 | **Set Up the SSH Agent** | Lets 1Password manage SSH keys (no private keys on disk) |
| 3 | **Display key names when authorizing connections** | Shows which key is being used for each SSH connection |
| 4 | **Check for developer credentials on disk** | Watchtower alerts for leaked keys/tokens found on disk |
| 5 | **Generate SSH config file with bookmarked hosts** | Auto-creates `~/.ssh/1Password/config` from SSH bookmarks |

### Settings > Security

| # | Setting | Purpose |
|---|---------|---------|
| 6 | **Check for vulnerable passwords** | Watchtower checks passwords against the Pwned Passwords database |

### Quick Access

Open 1Password directly to Developer settings:

```bash
open "onepassword://settings/developer"
```

### Verification

After enabling the SSH agent:

```bash
ssh-add -l               # Should list your 1Password SSH keys
ssh -T git@github.com    # Should authenticate via 1Password
```

## SSH Agent

Once enabled, the dotfiles configure:

- **`~/.ssh/config`** — Points `IdentityAgent` to the 1Password agent socket
- **`~/.config/1Password/ssh/agent.toml`** — Controls which keys the agent offers (all by default)
- **`~/.ssh/1Password/config`** — Auto-generated by 1Password from SSH bookmarks (when setting #5 is enabled)

### Pinning Keys to Hosts

Edit `~/.config/1Password/ssh/agent.toml` to restrict which keys are offered:

```toml
# Only offer a specific key for GitHub
[[ssh-keys]]
item = "GitHub SSH Key"
vault = "Development"

# Pin a key to a host pattern
[[ssh-keys]]
item = "Work SSH Key"
vault = "Work"
host = "*.internal.company.com"
```

See: [1Password SSH Agent Config](https://developer.1password.com/docs/ssh/agent/config/)

## Usage in Templates

Reference secrets in `.tmpl` files using chezmoi's 1Password functions:

```
# Read a specific field
{{ onepasswordRead "op://vault/item/field" }}

# Example: API token
export GITHUB_TOKEN="{{ onepasswordRead "op://Development/GitHub Token/credential" }}"
```

## What Should Be Stored in 1Password

- SSH private keys
- API tokens and credentials
- Database passwords
- Service account keys

## What Should NOT Be in 1Password

- Public configuration (use regular template data)
- Non-sensitive defaults (use `.chezmoidata/`)
- Anything that needs to work without 1Password access

## Disabling 1Password

If you don't use 1Password, answer `false` when prompted during `chezmoi init`. Templates that reference 1Password are gated behind the `use_1password` data flag, and SSH config falls back to default behavior.

## Alternative: age Encryption

For file-level encryption without an external service, chezmoi also supports [age](https://age-encryption.org/):

```toml
# In chezmoi.toml
encryption = "age"

[age]
    recipient = "age1..."
```

Then add files with encryption:

```bash
chezmoi add --encrypt ~/.ssh/id_ed25519
```
